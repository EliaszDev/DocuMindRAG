What is Retrieval-Augmented Generation (RAG)?

RAG is a technique that combines information retrieval with text generation. Instead of relying solely on the parametric knowledge stored in a large language model (LLM), RAG first retrieves relevant documents from an external knowledge base and then passes them as context to the LLM.

This approach has several advantages:
1. It reduces hallucinations because the model is grounded in retrieved facts.
2. It allows the model to answer questions about private or recent data that was not in its training set.
3. It makes the reasoning process more transparent since you can inspect which documents were retrieved.

A typical RAG pipeline consists of four stages:
- Ingestion: loading and parsing source documents.
- Chunking: splitting documents into smaller pieces.
- Embedding & Storage: converting chunks into dense vectors and storing them in a vector database.
- Retrieval & Generation: finding the most relevant chunks for a query and synthesizing an answer.

LangChain provides modular components for each stage, making it easy to swap vector stores, embedding models, or LLMs without rewriting the entire pipeline.
